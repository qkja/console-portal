## 1. 需求概述

### 1.1 项目背景

console-portal 前端目前使用 mock 数据进行开发和演示（详见 REQ-002），所有界面和交互已开发完成。现需将前端数据从 mock 切换为对接 openplatformsvr 的**真实后端 API**，实现后端联调。

本次对接范围**仅限租户管理模块**（对应 REQ-002 中 T01~T06 功能），用户管理模块的 API 对接留待下期进行。

### 1.2 目标

1. 实现环境配置，支持开发/测试/生产环境无缝切换后端 API 地址
2. 将租户管理页面的 mock 数据全部替换为真实后端 API 调用
3. 后端 API 字段与前端 mock 字段不一致时，前端做字段映射处理
4. 所有接口请求失败时通过 Toast 显示错误信息
5. 保持现有的 UI 交互和布局**完全不变**

### 1.3 前提条件

- 后端 openplatformsvr 已部署运行，环境中可访问
- 后端 API 接口已稳定，返回格式符合规范
- 前端项目已配置好构建工具链

## 2. 接口详情

### 2.1 通用约定

| 项 | 说明 |
|----|------|
| 后端地址 | `http://localhost:8080`（暂定，通过环境变量 VITE_API_BASE_URL 配置） |
| 鉴权方式 | Header 传参，无需 Cookie/Token |
| 请求 Header | `X-User-Id` + `X-Tenant-Id`（统一由请求封装层注入） |
| 响应格式 | JSON，遵循后端 openplatformsvr 规范 |
| Content-Type | `application/json` |

### 2.2 租户管理 API 接口（本期对接）

基础路径：`/open/api/v1/tenants`

| # | 方法 | 路径 | 说明 | 对应前端功能 |
|:-:|------|------|------|-------------|
| 1 | GET | `/open/api/v1/tenants` | 查询租户列表（分页+搜索+状态筛选） | 租户列表展示（T01） |
| 2 | POST | `/open/api/v1/tenants` | 注册/创建租户 | 创建租户（T02） |
| 3 | PUT | `/open/api/v1/tenants/:tenantId` | 更新租户信息 | 编辑授权（T03） |
| 4 | DELETE | `/open/api/v1/tenants/:tenantId` | 删除租户 | 删除租户（T04） |
| 5 | GET | `/open/api/v1/tenants/:tenantId` | 查询租户详情 | 查看详情（T05） |
| 6 | PUT | `/open/api/v1/tenants/:tenantId/freeze` | 冻结租户 | 备用（后续对接） |
| 7 | PUT | `/open/api/v1/tenants/:tenantId/unfreeze` | 解冻租户 | 备用（后续对接） |

### 2.3 详细接口说明

#### 2.3.1 查询租户列表

```
GET /open/api/v1/tenants
```

**请求参数（Query String）：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| page | integer | 否 | 页码（默认 1） |
| pageSize | integer | 否 | 每页条数（默认 10，最大 100） |
| keyword | string | 否 | 搜索关键词（模糊匹配租户名称） |
| status | integer | 否 | 状态筛选（0=已授权, 1=未授权，其他值不过滤） |

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "t_xxxxx",
        "name": "XX科技有限公司",
        "contactName": "张三",
        "contactPhone": "13800138000",
        "status": 0,
        "authExpireAt": "2026-12-31T23:59:59+08:00",
        "createdAt": "2026-01-01T10:00:00+08:00",
        "updatedAt": "2026-05-01T10:00:00+08:00"
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

**字段映射说明：**

| 前端字段 | 后端字段 | 映射规则 |
|----------|----------|----------|
| id | id | 直接映射 |
| name | name | 直接映射 |
| contactPerson | contactName | 字段名不同，前端做映射 |
| contactPhone | contactPhone | 直接映射 |
| authStatus | status | 0=已授权, 1=未授权（前端展示文案映射） |
| authExpire | authExpireAt | 字段名不同，前端做映射 |
| createdAt | createdAt | 直接映射 |
| updatedAt | updatedAt | 直接映射 |

#### 2.3.2 创建租户

```
POST /open/api/v1/tenants
```

**请求体：**

```json
{
  "name": "XX科技有限公司",
  "contactName": "张三",
  "contactPhone": "13800138000"
}
```

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "t_xxxxx"
  }
}
```

**前端逻辑：** 创建成功后，前端弹出「授权配置」弹窗让用户配置授权信息，然后调用编辑租户接口设置授权详情。

#### 2.3.3 更新租户信息

**请求体：**

```json

  "message": "success",
  "data": {}
}
```

#### 2.3.4 删除租户

```

**响应示例：**

  "message": "success",

```

#### 2.3.5 查询租户详情

```

```

**响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "t_xxxxx",
    "name": "XX科技有限公司",
    "contactName": "张三",
    "contactPhone": "13800138000",
    "status": 0,
    "authExpireAt": "2026-12-31T23:59:59+08:00",
    "createdAt": "2026-01-01T10:00:00+08:00",
    "updatedAt": "2026-05-01T10:00:00+08:00"
  }

```

### 2.4 错误响应（通用）

所有接口在出错时返回以下格式：

```json
{
  "code": -1,
  "message": "错误描述信息",
  "data": null
}
```

| code | 说明 | 前端处理 |
|:----:|------|----------|
| 0 | 成功 | 正常处理数据 |
| 400 | 参数错误 | Toast 显示 message |
| 401 | 无权限 | Toast 提示 "无访问权限" |
| 404 | 资源不存在 | Toast 提示 "资源不存在" |
| -1 | 系统错误 | Toast 显示 message |
| 其他 | 未知错误 | Toast 显示 "操作失败，请稍后重试" |

## 3. 需求详述

### 3.1 环境配置（编号 001）

**需求描述：** 支持通过环境变量切换后端 API 地址，实现开发、测试、生产环境的无缝切换。

**验收标准：**
- [x] 项目根目录 `.env` 文件中增加 `VITE_API_BASE_URL` 变量，默认值为 `http://localhost:8080`
- [x] `.env.development` 设置开发环境值
- [x] `.env.production` 设置生产环境值
- [x] 前端请求工具函数读取 `VITE_API_BASE_URL` 作为基础路径
- [x] 构建时自动按环境注入对应值，无需修改代码

### 3.2 请求封装（编号 002）

**需求描述：** 封装统一的 HTTP 请求工具函数，自动注入鉴权 Header 和基础路径。

**验收标准：**
- [x] 基于 axios 或原生 fetch 封装统一的 `request` 函数
- [x] 自动拼接 `VITE_API_BASE_URL` + 接口路径
- [x] 自动注入 Header `X-User-Id` 和 `X-Tenant-Id`（值从环境变量或配置读取，初期可写死占位值）
- [x] 统一响应拦截，提取 `response.data` 层
- [x] 统一错误拦截，抛出错误信息供业务层处理
- [x] Content-Type 默认为 `application/json`

### 3.3 租户列表对接后端（编号 003）

**需求描述：** 将租户列表的数据源从 mock 改为调用后端 GET `/open/api/v1/tenants` 接口。

**验收标准：**
- [x] 列表初次加载时调用后端 API 获取数据
- [x] 分页切换、搜索、状态筛选均调用后端 API
- [x] 列表字段按 2.3.1 节字段映射表做映射处理
- [x] 分页信息（total, page, pageSize）来自后端响应
- [x] 列表为空时展示空状态（保持现有 UI）
- [x] 请求过程中展示 loading 状态（保持现有 UI）
- [x] 保持现有表格列、操作按钮、筛选条件布局不变

### 3.4 创建租户对接后端（编号 004）

**需求描述：** 将创建租户的数据提交从 mock 改为调用后端 POST `/open/api/v1/tenants` 接口。

**验收标准：**
- [x] 创建租户表单提交时调用后端 API
- [x] 创建成功 → 弹出「授权配置」弹窗（保持现有交互）
- [x] 授权配置提交 → 调用编辑接口 PUT `/open/api/v1/tenants/:tenantId` 保存授权信息
- [x] 全部操作成功 → 刷新列表
- [x] 接口返回错误 → Toast 显示错误信息，不关闭弹窗（用户可修正后重试）

### 3.5 编辑租户对接后端（编号 005）

**需求描述：** 将编辑租户的数据提交从 mock 改为调用后端 PUT `/open/api/v1/tenants/:tenantId` 接口。

**验收标准：**
- [x] 编辑弹窗提交时调用后端 API
- [x] 提交参数包含所有编辑的字段（名称、联系人、授权到期时间等）
- [x] 编辑成功 → 自动刷新列表
- [x] 接口返回错误 → Toast 显示错误信息，不关闭弹窗

### 3.6 删除租户对接后端（编号 006）

**需求描述：** 将删除租户确认后的操作从 mock 改为调用后端 DELETE `/open/api/v1/tenants/:tenantId` 接口。

**验收标准：**
- [x] 删除确认后调用后端 API
- [x] 删除成功 → 自动刷新列表
- [x] 接口返回错误 → Toast 显示错误信息

### 3.7 租户详情对接后端（编号 007）

**需求描述：** 将租户详情的数据源从 mock 改为调用后端 GET `/open/api/v1/tenants/:tenantId` 接口。

**验收标准：**
- [x] 点击「查看详情」时调用后端 API 获取数据
- [x] 详情页显示前端字段按 2.3.5 节映射
- [x] 保持现有详情弹窗布局不变
- [x] 请求过程中展示 loading 状态
- [x] 接口失败时显示错误信息

### 3.8 错误处理（编号 008）

**需求描述：** 所有 API 请求失败时，在界面上通过 Toast 提示用户错误信息。

**验收标准：**
- [x] 请求超时、网络错误、HTTP 错误码均在拦截层捕获
- [x] Toast 提示内容优先使用后端返回的 `message` 字段
- [x] 后端未返回具体 message 时使用通用文案
- [x] Toast 类型为错误样式（红色/警告图标）
- [x] Toast 展示时长 3~5 秒
- [x] 不影响当前页面状态（之前已加载的数据正常展示）

### 3.9 授权字段映射（编号 009）

**需求描述：** 后端 API 字段与前端 mock 字段不一致时，前端做字段映射处理。后端有的字段直接映射，后端没有的字段使用默认值。

**验收标准：**
- [x] 按 2.3.1 节字段映射表实现前端映射层
- [x] 映射逻辑集中管理（推荐使用 transform 函数或 map 配置）
- [x] 后端返回的新增字段不会导致前端报错
- [x] 后端缺少的字段取默认值（如空字符串、0、null）

## 6. 风险评估

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|:----:|----------|
| 后端接口返回值格式与预期不符 | 前端解析失败 | 中 | 开发阶段提前联调确认，mock 数据对齐后端格式 |
| 后端接口延迟高或不可用 | 页面加载缓慢或不可用 | 低 | 统一请求拦截层设置超时（默认 10s），超时友好提示 |
| 后端鉴权 Header 验证失败 | 全部接口报 401 | 低 | 请求封装层固定注入 X-User-Id 和 X-Tenant-Id，联调时确认值正确 |
| 字段映射遗漏 | 部分字段显示空值 | 低 | 映射层兜底处理，缺失字段使用默认值 |
| 接口返回大量数据 | 列表渲染卡顿 | 低 | 后端已做分页，前端按每页 10 条请求 |
