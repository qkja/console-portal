## 1. 需求概述

### 1.1 项目背景

基于现有的 openplatformsvr（Go 后端 API 网关），需要开发两个前端管理界面，为平台运营人员和租户管理员提供可视化的管理操作能力。

现有后端 API 已覆盖完整的多租户管理与用户管理能力，本需求**仅关注前端页面实现**，不涉及后端接口的改动。

### 1.2 目标

- 为租户管理员提供本租户下的用户管理能力（增删改查、状态管理）
- 为超级管理员提供平台级租户管理能力（增删改查、审核、状态管理）
- 界面简洁易用，交互流畅，符合后台管理系统通用规范
- 第一期实现两个独立页面，后续可扩展更多功能模块

### 1.3 目标用户

| 角色 | 说明 | 使用场景 |
|------|------|----------|
| 超级管理员 | 平台运营方，管理所有租户 | 租户列表管理、租户审核、租户状态管理 |
| 租户管理员 | 租户内部管理员，管理本租户的用户 | 本租户用户增删改查、锁定/解锁、重置密码 |

### 1.4 设计原则

1. **遵循现有 UI 规范** — 与 openplatformsvr 已有的管理后台风格保持一致
2. **组件化开发** — 列表、弹窗、表单、状态标签等均复用统一组件
3. **操作反馈即时** — 所有操作成功后即时刷新列表，失败时明确提示错误原因
4. **权限感知** — 页面层面的按钮可见性应与用户角色权限对应（超级管理员 vs 租户管理员）

## 2. 界面一：租户用户管理

### 2.1 页面概述

- **页面标题**：用户管理
- **访问路径**：`/tenant/users`（示例路由，具体以项目实际路由为准）

- **页面入口**：租户管理后台侧边导航栏 → 「用户管理」

### 2.3 目录管理

#### 2.3.1 目录说明

- 每个租户创建后**自动生成一个默认「本地目录」**
- 租户管理员可以新建其他目录，如：企微目录、钉钉目录、飞书目录（后续用于第三方数据同步）
- 每个目录是一个独立的数据源入口
- 用户管理界面的数据来自**当前选中的目录**
- 页面顶部有目录切换器：`[本地目录 ▼]` 下拉选择

### 2.4 功能列表

#### 2.4.1 用户列表展示 (U01)

| 展示字段 | 类型 | 说明 |
|----------|------|------|
| 用户名 | 文本 | 用户登录账号，不可重复 |
| 真实姓名 | 文本 | 用户真实姓名 |
| 手机号 | 文本 | 脱敏显示中间四位（如 138****1234） |
| 邮箱 | 文本 | 完整显示 |
| 所属组织 | 文本 | 格式：目录名/部门名/组名 |
| 状态 | 标签 | 🟢 正常（绿色标签）、🔴 锁定（红色标签）、🟡 未激活（黄色标签） |
| 创建时间 | 时间 | 格式：YYYY-MM-DD HH:mm |

- **表格交互**：支持按任意列头排序（点击列头切换升序/降序/无排序）
- **空状态**：无数据时显示「暂无用户数据」空状态插画
- **加载状态**：加载中显示骨架屏或 Loading 动画

#### 2.3.2 用户搜索与筛选 (U07)

- **搜索框**：支持按用户名、真实姓名模糊搜索
- **状态筛选**：下拉选择「全部 / 正常 / 锁定 / 未激活」
- **组合筛选**：搜索关键词与状态筛选可以同时生效
- **清空筛选**：筛选条件旁显示「重置」按钮，一键清空所有筛选条件

#### 2.3.3 创建用户 (U02)

- **触发方式**：点击页面右上角「新建用户」按钮 → 弹出 Modal 表单
- **表单字段**：

| 字段 | 类型 | 必填 | 限制 |
|------|------|:----:|------|
| 用户名 | 输入框 | ✅ | 3~32 位字母数字下划线组合 |
| 真实姓名 | 输入框 | ✅ | 2~20 个字符 |
| 手机号 | 输入框 | ✅ | 11 位手机号格式校验 |
| 邮箱 | 输入框 | ✅ | 邮箱格式校验 |
| 所属目录 | 下拉选择 | ✅ | 选择该用户归属的目录 |
| 所属组织 | 级联选择 | ✅ | 选择目录下的组织架构节点 |
| 所属用户组 | 下拉选择 | ❌ | 可选，选择组织下的用户组 |
| 初始密码 | 输入框 | ❌ | 不填则生成随机密码 |

- **交互细节**：
  - 选择「所属目录」后，自动加载该目录下的组织架构树供选择
  - 选择「所属组织」后，自动加载该组织下的用户组供选择
  - 所有字段实时校验，错误提示显示在字段下方
- **提交行为**：点击「确认」→ 显示加载状态 → 创建成功后关闭弹窗 → 刷新列表 → Toast 提示「创建成功」
- **取消行为**：点击「取消」或弹窗右上角 ✕ → 关闭弹窗，**不保存**已填写内容
- **异常处理**：创建失败时弹窗不关闭，在弹窗顶部显示红色错误提示，保留已填内容允许用户修正后重试

#### 2.3.4 编辑用户 (U03)

- **触发方式**：点击行操作列「编辑」按钮 → 弹出 Modal 表单（复用创建表单组件，预填当前用户数据）
- **可编辑字段**：真实姓名、手机号、邮箱、所属组织、所属用户组
- **不可编辑字段**：用户名（灰化显示，不允许修改）
- **交互细节**：与创建表单相同组件，字段预填当前值
- **提交行为**：点击「保存」→ 更新成功后关闭弹窗 → 刷新列表 → Toast 提示「更新成功」

#### 2.3.5 删除用户 (U04)

- **触发方式**：点击行操作列「删除」按钮
- **二次确认**：弹出确认对话框
  ```
  ┌─────────────────────────┐
  │  确认删除用户            │
  │                          │
  │  确定要删除用户「张三」  │
  │  吗？此操作不可撤销。    │
  │                          │
  │     [取消]    [确认删除]  │
  └─────────────────────────┘
  ```
- **提交行为**：点击「确认删除」→ 删除成功后关闭弹窗 → 刷新列表 → Toast 提示「删除成功」
- **异常处理**：若该用户存在关联数据无法删除，弹窗显示具体错误原因
- **权限控制**：超级管理员与租户管理员均有删除权限，但**不可删除自己**

#### 2.3.6 锁定/解锁用户 (U05)

- **触发方式**：
  - 当前用户状态为「正常」→ 操作列显示「锁定」按钮
  - 当前用户状态为「锁定」→ 操作列显示「解锁」按钮
- **二次确认**：
  - 锁定确认：`确定要锁定用户「用户名」吗？锁定后用户将无法登录系统。`
  - 解锁确认：`确定要解锁用户「用户名」吗？解锁后用户可正常登录。`
- **提交行为**：确认后 → 调用状态变更接口 → 刷新列表 → Toast 提示「已锁定/已解锁」
- **状态切换**：锁定成功后，该行状态标签即时更新为「锁定」，操作按钮切换为「解锁」

#### 2.3.7 重置密码 (U06)

- **触发方式**：点击行操作列「重置密码」按钮
- **二次确认**：
  ```
  ┌─────────────────────────────────┐
  │  重置密码                        │
  │                                  │
  │  将为用户「用户名」重置密码。    │
  │  重置后请及时通知用户修改密码。  │
  │                                  │
  │     [取消]    [确认重置]         │
  └─────────────────────────────────┘
  ```
- **提交行为**：确认后 → 调用重置密码接口 → 弹窗显示新密码（**一次性展示，不可再次查看**）
  ```
  ┌─────────────────────────────────┐
  │  密码重置成功                    │
  │                                  │
  │  新密码: AbCd1234XyZ            │
  │                                  │
  │  请及时告知用户并修改密码。      │
  │                                  │
  │         [我知道了]               │
  └─────────────────────────────────┘
  ```
- **安全要求**：新密码仅此弹窗展示一次，关闭后无法再次查看。如需再次查看需重新执行重置操作

### 2.5 验收标准

| # | 验收项 | 通过条件 |
|:-:|--------|----------|
| 1 | 用户列表展示 | 列表正确展示所有字段，状态标签颜色正确 |
| 2 | 搜索筛选 | 按用户名/姓名搜索、按状态筛选正常，组合筛选正常 |
| 3 | 排序 | 点击各列头可正常排序 |
| 4 | 分页 | 分页控件正常，每页条数切换正常 |
| 5 | 创建用户 | 表单校验正常，必填项有提示，创建成功后列表刷新，新用户可见 |
| 6 | 编辑用户 | 表单预填正常，编辑成功后列表数据更新 |
| 7 | 删除用户 | 二次确认弹窗正常，删除后列表更新，后端同步删除 |
| 8 | 锁定/解锁 | 按钮根据状态正确显示，锁定后用户无法登录，解锁后恢复 |
| 9 | 重置密码 | 重置成功后弹窗显示新密码，新密码可正常登录 |
| 10 | 空状态 | 无数据时正确显示空状态提示 |
| 11 | 加载状态 | 数据加载中显示 Loading 状态 |
| 12 | 错误处理 | 接口返回错误时，Toast 显示错误提示，不影响其他数据 |

```
│  ☑ 日志审计授权     操作审计     │
│                                   │
│  ⚠ 变更授权会影响该租户下        │
│   所有用户的系统权限              │
│                                   │
│     [取消]          [保存修改]    │
└──────────────────────────────────┘
```

- **可编辑字段**：联系人、联系电话、联系邮箱、授权勾选
- **不可编辑字段**：租户名称、租户编码
- **提交行为**：点击「保存」→ 更新成功后关闭弹窗 → 刷新列表 → Toast「授权已更新」
- **提示**：授权变更区域上方显示警示文案

- **二次确认**：

### 3.4 验收标准

| # | 验收项 | 通过条件 |

## 6. 全局交互规范

### 4.1 通用组件定义

| 组件 | 规格 | 说明 |
|------|------|------|
| 表格 | Ant Design Table / Element UI Table | 斑马纹、hover 高亮、固定操作列 |
| 弹窗（Modal） | 居中显示 | 宽度 520px~720px，带遮罩层，点击遮罩不关闭 |
| 下拉选择 | 支持搜索 | 数据量超过 10 条时提示「请输入搜索关键词」 |
| 级联选择 | 组织架构树 | 多层级结构，支持搜索节点 |
| Toast 提示 | 右上角弹出 | 成功-绿色、错误-红色、警告-黄色，3 秒自动消失 |
| 确认弹窗 | 居中显示 | 标题 + 描述文案 + [取消] [确认] 按钮 |
| 状态标签 | 圆角标签 | 不同状态不同颜色，配文字描述 |
| 骨架屏 | 数据加载中 | 表格加载时显示 5 行骨架屏 |
| 空状态 | 居中插画 | 插画 + 文案提示 |
| 分页 | 底部居中 | 显示总条数、页码、每页条数选择器 |

### 4.2 操作反馈规范

| 操作类型 | 反馈方式 | 说明 |
|----------|----------|------|
| 查询/搜索 | 列表刷新 | 加载中显示骨架屏，完成后更新表格 |
| 新增/编辑 | 关闭弹窗 + Toast + 刷新列表 | 保持当前页码 |
| 删除 | Toast + 刷新列表 | 若当前页仅剩一条且非首页，自动跳转上一页 |
| 状态变更 | Toast + 刷新列表 | 即时更新状态标签和操作按钮 |
| 接口错误 | Toast 错误提示 | 显示后端返回的具体错误信息 |

### 4.3 权限控制

| 界面 | 可查看角色 | 可操作角色 |
|------|------------|------------|
| 租户用户管理 | 租户管理员 | 租户管理员 |
| 所有租户管理 | 超级管理员 | 超级管理员 |

> 前端通过 JWT Token 中的角色信息控制页面可见性和按钮可点击性。
> 租户管理员不应看到「所有租户管理」页面的入口。

### 4.4 前端请求规范

- **请求方式**：所有接口请求鉴权头统一为 `Authorization: Bearer <token>`
- **请求/响应格式**：JSON
- **错误码处理**：统一拦截 HTTP 4xx/5xx 状态码，在 Toast 中展示 `message` 字段内容
- **Token 过期**：Token 过期时自动跳转到登录页

## 5. 依赖的后端 API

> **注意**：以下接口 openplatformsvr 已实现，前端只需对接调用，无需改动后端代码。

### 5.1 用户管理接口（租户用户管理界面使用）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tenants/{tenantId}/users` | 查询租户用户列表 |
| POST | `/api/tenants/{tenantId}/users` | 创建用户 |
| GET | `/api/tenants/{tenantId}/users/{userId}` | 查询用户详情 |
| PUT | `/api/tenants/{tenantId}/users/{userId}` | 更新用户信息 |
| DELETE | `/api/tenants/{tenantId}/users/{userId}` | 删除用户 |
| PATCH | `/api/tenants/{tenantId}/users/{userId}/status` | 锁定/解锁用户 |
| POST | `/api/tenants/{tenantId}/users/{userId}/reset-password` | 重置用户密码 |
| GET | `/api/tenants/{tenantId}/directories` | 查询目录列表 |
| GET | `/api/tenants/{tenantId}/directories/{dirId}/orgs` | 查询组织架构树 |
